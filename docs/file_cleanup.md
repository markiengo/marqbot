# File Cleanup: Real Safe Checklist

Use this before deleting or archiving files in this repo.
Goal: remove clutter without breaking MarqBot.

## 1) Start safe
- Run `git status`.
- Write down:
  - what paths are in scope
  - what paths are out of scope
  - what checks you will run after cleanup
- If scope is unclear, do not delete anything.

## 2) Know what is protected
Do not delete these without clear proof and explicit approval:
- `docs/`
- `.claude/`
- `todo.md`
- `marquette_courses_full.xlsx`
- `data/`
- `render.yaml`
- `.github/workflows/`
- `infra/`

Also remember:
- most docs are local-only
- inside `docs/`, only `docs/CHANGELOG.md` and `docs/data_model.md` are meant to be pushable

## 3) Good cleanup candidates
These are usually safe local cleanup targets:
- `.pytest_cache/`
- `**/__pycache__/`
- `.next/`
- `frontend/out/`
- `coverage/`
- `*.log`
- `*.tmp`

These are not automatically safe:
- scripts
- tests
- generated frontend files
- old CSVs or workbook files
- anything under `backend/`, `frontend/src/`, or `data/`

## 4) Prove a file is unused
Before deleting a real file, check all of these:
- runtime code
- tests
- docs
- scripts
- workflows

Use repo-wide search such as:
- `rg -n "name_or_path" backend frontend tests scripts docs .github`

Keep the file if:
- it is referenced anywhere
- it is generated by a script you still use
- it is part of deploy or local run flow
- you are not sure

## 5) MarqBot-specific danger areas
Be extra careful with:
- `backend/server.py`
- `backend/semester_recommender.py`
- `backend/eligibility.py`
- `frontend/src/lib/api.ts`
- `frontend/src/lib/types.ts`
- `frontend/src/lib/quips.ts`
- `frontend/src/lib/quipBank.generated.ts`
- `scripts/compile_quips.py`
- all CSVs in `data/`

These files connect to a lot of behavior and are easy to break by accident.

## 6) Decide: delete, archive, or keep
- `delete_now`
  - clear proof the file is unused
- `archive_for_review`
  - probably unused, but not worth risking today
- `keep`
  - referenced, generated, or unclear

If unsure, use `keep`.

## 7) Run checks after cleanup
Pick the checks that match the cleanup:

### Backend-related cleanup
- `python -m pytest tests/backend -q`

### Frontend-related cleanup
- `cd frontend && npm run test`
- `cd frontend && npm run build`

### Quip cleanup
- `python scripts/compile_quips.py`

### Workflow cleanup
- read the workflow file carefully
- make sure paths and commands still exist

## 8) Report what happened
Always write down:
- deleted files
- archived files
- kept files that looked suspicious but were not safe to remove
- commands you ran
- any risk still left

Use this format:
- `Deleted`: path | reason
- `Archived`: path | reason
- `Kept`: path | reason
- `Validation`: command | result
- `Risk`: what still needs manual review

## 9) Simple rule
If you do not have strong proof, do not delete the file.
