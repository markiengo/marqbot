# Requirements: MarqBot v1.0 UI Upgrade

**Defined:** 2026-03-29
**Core Value:** MarqBot must give students clear, trustworthy next-course guidance grounded in the actual degree rules.

## v1 Requirements

### Hero Stage

- [ ] **HERO-01**: Visitor sees the hero occupy the first desktop viewport without the next section competing for attention
- [ ] **HERO-02**: Visitor sees a centered MarqBot mockup as the dominant focal object on first load
- [ ] **HERO-03**: Visitor can reach the primary onboarding CTA from the hero without relying on dense supporting copy
- [ ] **HERO-04**: Visitor sees an intentional scroll cue or continuation hint that clearly invites the first scroll

### Scroll Experience

- [ ] **FLOW-01**: Visitor experiences a deliberate handoff from the hero into section two instead of an accidental overlap between sections
- [ ] **FLOW-02**: Visitor sees one signature unfold or pinned transition that turns the hero mockup into supporting feature storytelling
- [ ] **FLOW-03**: Visitor sees only major sections use dramatic motion while secondary content stays calmer and easier to parse

### Feature Story

- [ ] **FEAT-01**: Visitor sees section two spotlight one dominant benefit at a time instead of three equal-weight cards
- [ ] **FEAT-02**: Visitor sees supporting chips, cards, or panels reinforce the active benefit without making the page feel cluttered
- [ ] **FEAT-03**: Visitor reaches a proof/trust beat after the spectacle moment that reinforces MarqBot's rule-based credibility

### System Fit

- [ ] **SYS-01**: Visitor sees the redesigned homepage remain recognizably MarqBot through the existing navy, gold, and blue visual language
- [ ] **SYS-02**: Visitor sees the how-it-works section and final CTA feel like part of the same new visual system instead of older holdovers

### Performance and Accessibility

- [ ] **PERF-01**: Visitor on standard laptop hardware experiences smooth hero and scroll animation without obvious jank
- [ ] **PERF-02**: Visitor with reduced motion or reduced effects preferences gets a lighter experience that preserves the story and hierarchy
- [ ] **PERF-03**: Visitor on mobile gets the same core narrative with simplified layers, simplified motion, and stable CTA/mockup sizing

## Future Requirements

### Advanced Presentation

- **ADV-01**: Visitor can preview additional mockup states beyond the main hero composition
- **ADV-02**: Homepage adapts proof or emphasis by audience segment or entry context

### Optimization

- **OPT-01**: Landing variants can be A/B tested for scroll depth and onboarding conversion
- **OPT-02**: Effect budgets can be tuned from collected real-user performance telemetry

## Out of Scope

| Feature | Reason |
|---------|--------|
| Backend recommendation changes | This milestone only reworks homepage presentation |
| Live embedded planner in hero | Hybrid mockup gives the right wow factor without turning the landing page into a mini-app |
| Pagewide cursor tracking | Too expensive and distracts from hierarchy |
| Multiple pinned scroll scenes | One signature moment is enough and safer for performance |
| `backdrop-filter` heavy glass | The existing design system already prefers cheaper fake-glass treatments |
| Broad copy rewrite or rebrand | The approved direction for this milestone is visual-first |
