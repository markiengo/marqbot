## Strengths

- The planner card system feels deliberate rather than boilerplate. The progress strip, semester rail, and embedded recommendation list read as one coherent interface.
- The filmstrip semester selector is a good visual upgrade. It makes term switching easier to scan than the old equal-width tab grid.
- The new compact course rows increase information density without collapsing readability.
- The landing story section has stronger reset behavior and better entrance motion when the section re-enters view.

## Top Fixes

1. Restore page-level scrolling below desktop breakpoints. The current fixed-height shell is too brittle for short mobile viewports and test coverage already flags this regression.
2. Bring back a clearer task-specific planner control for schedule-style changes and profile edits. The generic `Settings` CTA hides what used to be explicit user actions.
3. Reduce the number of simultaneous glow and gradient layers in the planner shell. The interface is stylish, but the current stack is near the point where accents stop guiding and start competing.

## Detailed Findings

### 1. Small-screen resilience dropped

The planner now forces a fixed viewport shell at the page and component level. That makes the layout feel premium on desktop, but it reduces the margin for small mobile screens, browser chrome changes, and accessibility zoom.

Relevant files:
- `frontend/src/app/planner/page.tsx`
- `frontend/src/app/globals.css`

Why it matters:
- The planner should tolerate constrained heights without trapping content.
- Current tests already encode that requirement and now fail against the new shell behavior.

### 2. Priority and profile controls are less explicit

The planner still exposes the underlying actions, but the move from dedicated action labels to a generic `Settings` entry point makes the interface less self-explanatory. A user who wants to change schedule style or profile data now has to infer that `Settings` is the right door.

Relevant files:
- `frontend/src/components/planner/PlannerLayout.tsx`
- `frontend/src/components/planner/ProfileModal.tsx`
- `frontend/src/components/planner/ProfileStyleTab.tsx`

Why it matters:
- The old labels described the task directly.
- The new label is broader but less informative, especially in a task-driven planner workflow.

### 3. Visual density is close to overload

The new planner direction is strong, but the shell now combines a heavy background gradient, multiple radial glows, glossy card surfaces, highlighted pills, and embedded panel gradients all at once. The result still looks good, but the visual budget is nearly exhausted.

Relevant files:
- `frontend/src/app/globals.css`
- `frontend/src/components/planner/PlannerLayout.tsx`
- `frontend/src/components/planner/RecommendationsPanel.tsx`

Why it matters:
- Premium UI depends on emphasis.
- When every layer glows, fewer elements feel truly important.
